# Plan: Forge CLI — Gap Analysis vs Hermes CLI & OpenClaw CLI

## Goal
Compare Forge CLI (`@hermes-agent-dk/hermes-forge-cli`) against Hermes CLI and OpenClaw CLI, identify gaps, and define the next phase of development.

## Current State

### Forge CLI (v1.1.0)
- Node.js/TypeScript, Commander, chalk, neo-blessed
- 24 CLI commands, 1 TUI view (4 sub-views)
- Brand package (colors, logo, theme)
- Config: JSON file via `conf` package
- Published: GitHub Packages only
- ~59 source files, ~2K LOC
- Tests: Vitest
- **Intended use:** DevOps tool for Forge platform (pack management, health, dev, MCP)

### Hermes CLI (v0.13.0)
- **Language:** Python 3.11+
- **Architecture:** Argparse + 70+ slash commands
- **TUI:** Ink (React) + TypeScript — full-featured
- **Plugin system:** 4-layer discovery (bundled→user→project→pip), YAML manifest + hooks
- **Skill system:** SKILL.md markdown files as composable expertise
- **Tool registry:** Self-registering, availability-gated, toolset composition
- **Gateway:** 20+ messaging platforms, agent cache, session management
- **Tests:** 17,000 across 900 files
- **Extensibility:** Plugin SDK, provider abstraction, skin engine
- **Intended use:** AI agent platform

### OpenClaw CLI (2026.5.12-beta.1)
- **Language:** TypeScript ESM, Node 22.12+
- **Architecture:** Commander + policy-based routing (14 fast-path, 25+ sub-CLI)
- **TUI:** `@earendil-works/pi-tui` + WebSocket to Gateway
- **Plugin system:** SDK (`@openclaw/plugin-sdk`), 80+ extensions
- **Fast-path:** Sub-second command execution by skipping plugin loading
- **Compile cache:** Self-respawn for consistent behavior
- **Gateway:** Telegram, Discord, Slack, WhatsApp, Matrix, etc.
- **Tests:** Vitest, co-located test files
- **Intended use:** AI agent platform (competitor to Hermes)

## Key Gaps (Forge CLI vs industry)

### 1. Plugin System (biggest gap)
- No way for third parties to extend Forge CLI
- No lifecycle hooks, no plugin discovery
- **Fix:** Plugin registry similar to Hermes' 4-layer discovery

### 2. Skill/Extension System
- No composable "modes" or expertise packs
- OpenClaw has extensions/; Hermes has skills/
- **Fix:** SKILL.md convention or extension directory

### 3. Fast-Path Architecture
- Forge CLI loads everything on every command
- OpenClaw's policy-based routing skips plugin loading for simple commands
- **Fix:** Lazy import loaders for heavy modules

### 4. TUI Quality
- neo-blessed is unmaintained (last update 2022)
- Hermes uses Ink (React, active development)
- **Fix:** Rewrite TUI to Ink

### 5. Gateway / Platform Integration
- No messaging platform integration
- Both Hermes and OpenClaw have Telegram/Discord/etc. gateways
- **Fix:** Gateway adapter for Forge platform notifications

### 6. Testing
- ~20 test files, no coverage target
- Hermes has 17K tests
- **Fix:** Add integration tests for remote commands

### 7. Distribution
- GitHub Packages only
- Should be on npmjs.org for discoverability

## What Forge CLI Does *Better*

1. **Simplicity** — 2K LOC vs 100K+ is easier to understand and modify
2. **Focused scope** — platform DevOps tool, not an AI agent platform
3. **Brand package** — Hermes/OpenClaw don't have a dedicated brand module
4. **Build speed** — TypeScript build in ~8 seconds

## Proposed Roadmap (Prioritized)

### Phase 1: Plugin System (~4h)
- `extensions/` directory with manifest.yaml convention
- Command injection: `forge <plugin-name> <subcommand>`
- Lifecycle hooks: `on_command`, `on_init`
- Registry: built-in > user > project

### Phase 2: Ink TUI (~6h)
- Rewrite from neo-blessed to Ink (React)
- Share React components with Forge web app
- Keyboard-driven, streaming responses

### Phase 3: Fast-Path (~2h)
- Lazy imports for heavy modules (MCP, remote, monitor)
- Sub-second startup for `forge status`, `forge --help`

### Phase 4: npmjs.org (~10min)
- Switch publishConfig to npmjs.org
- Requires npm token

### Phase 5: Tests (~3h)
- Integration tests for remote commands
- Coverage target: 60%+
- E2E smoke tests for TUI

## Assumptions
- Forge CLI stays a **DevOps/tooling CLI**, not an AI agent platform
- Plugin system follows OpenClaw's extension model (simpler than Hermes')
- Ink TUI shares design tokens with Forge web app
- User provides npm token when ready for npmjs.org publish

## Files Likely to Change (Phase 1)
- `src/commands/plugin.ts` — Plugin management commands
- `src/lib/pluginManager.ts` — Plugin loader (rewrite)
- `src/types.ts` — Plugin manifest types
- `extensions/` — NEW directory

## Risks
- Neo-blessed is already installed and working; Ink rewrite is a large effort
- Plugin system adds surface area for security concerns
- Fast-path requires refactoring imports across 24 commands
